About the Project

Predictive Maintenance for EVs — Failure Probability & Smart Service

EVIoT Predict transforms raw EV telemetry into early warnings. It estimates Failure Probability, suggests Maintenance Type, and provides RUL/TTF guidance.

What it predicts

• Failure Probability (binary risk) • Maintenance Type (multiclass) • RUL — Remaining Useful Life (regression) • TTF — Time To Failure (regression)

Designed to improve uptime, safety, and service planning.

Tech Stack

Flask • Scikit‑learn / XGBoost • Pandas • NumPy • HTML/CSS/JS

Modular pipelines for preprocessing, training, and inference.

Dataset

EVIoT‑PredictiveMaint dataset: IoT‑enabled EVs, 15‑minute intervals over 5 years. 8+ feature groups covering load, temperature, vibration, usage patterns, and environment.

Curated for predictive maintenance research and production prototypes.

Method

Feature engineering • Class rebalancing • Model selection (RF, XGB, etc.) • Cross‑validation • Calibration. Separate pipelines for classification and regression targets with tracked metrics.

Deployable via REST endpoints in Flask.

Live Risk Scoring

Compute risk from real‑time telemetry to prevent sudden breakdowns.

Smart Maintenance

Recommends maintenance type and timing using learned patterns.

Explainability

Feature importances and trend views to justify predictions.

Model Metrics (sample)

Failure (binary): Accuracy ~90% • Precision/Recall tracked with class balance.

Confusion matrix & ROC‑AUC available in the dashboard.

Responsible AI

Data privacy by design. Monitoring for drift. Human‑in‑the‑loop for critical actions.

Supports audit logs and opt‑outs.
