About the Project

Predictive Maintenance for EVs — Failure Probability Only

EVIoT Predict transforms EV telemetry into early warnings. Our model estimates the probability of failure (binary) for critical components so fleets can act before issues become breakdowns.

What it does

Single-Task Focus: Failure Probability (Binary)

Clear scope, reliable outcomes.

  • Predicts whether a component is likely to fail within a defined horizon (e.g., next N days/cycles).
  • Designed for fleets needing practical, early warnings rather than complex multi-task outputs.
  • Outputs calibrated probability to support threshold-based actions (e.g., schedule inspection at ≥ 0.7).
View Technology Get in Touch
Quick facts
  • Telemetry granularity: 15 minutes
  • Time span: ~5 years
  • Typical features: temp, voltage, current, vibration, usage
  • Serving: REST (Flask)
Technology

Model & Pipeline

  • Problem framing: Binary classification for failure vs. no-failure.
  • Algorithms: XGBoost / Random Forest / Logistic Regression (configurable).
  • Preprocessing: Missing-value handling, outlier capping, feature scaling/encoding.
  • Class imbalance: Class weights or SMOTE (optional) based on validation.
  • Validation: Time-aware splits; metrics: ROC-AUC, PR-AUC, F1, calibration error.
  • Calibration: Platt scaling / isotonic for trustworthy probabilities.
Data & Serving

Data Flow

  • Ingestion: Batch CSV/Parquet or streaming IoT gateway.
  • Feature groups: Electrical (V, I), thermal (temps), vibration, usage cycles, environment.
  • Serving: Flask REST API: POST /predict → returns { probability, label }.
  • Monitoring: Data drift checks, rolling AUC/precision, alerting hooks.
